## 今日经验教训

### Desktop 开发经验

| # | 问题 | 教训 |
|---|------|------|
| 1 | `window.omc` API 在 Vite dev 模式不存在 | **必须提供 fallback 数据**，不要假设 API 始终可用 |
| 2 | CDP `dispatchKeyEvent` 模拟键盘有竞态 | 部分字符会丢失，这是浏览器限制，非代码 bug |
| 3 | Chrome 重启后 CDP port 会变 | 每次测试前需重新启动带 `--remote-debugging-port=9222` 的 Chrome |
| 4 | Browser 工具无法访问 localhost | 改用 CDP 直连 Chrome 进行测试 |
| 5 | App.tsx 使用内联 ConfigPanel 而非 SettingsPanel | 检查组件导入和使用是否一致，不要假设已集成 |

### 代码质量

- 修改组件后必须验证实际渲染效果，不要只看代码
- 添加 fallback 机制时，要测试 fallback 路径确实能工作
- 截图验证是最可靠的 UI 测试方式

### Git 工作流

- 每完成一个组件立即 commit + push，不攒
- 多个相关改动可以合并到一个 commit（如 P1-2 和 P1-3）
- Commit message 要说明"做了什么"和"为什么"
